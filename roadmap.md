# Frontend roadmap  
## Developer focus planner and execution log

---

## Phase 0: Foundation

- decide final product name
- decide color theme and typography
- create basic wireframe
- define folder structure

---

## Phase 1: Core architecture

- initialize html, css, and vanilla js project
- create global state object
- implement localStorage persistence layer
- build base layout and navigation
- implement header with persistent timer indicator

---

## Phase 2: Weekly planning

- weekly view with days
- add planned focus sessions per day
- add basic todos or events
- implement editing rules
- enforce 40 percent edit limit for current day

---

## Phase 3: Focus sessions

- custom duration input
- optional minimum success time
- start, pause, resume, end logic
- dedicated focus page
- notes support
- global timer persistence

---

## Phase 4: Video focus feature

- optional video input
- distraction free embed
- pause sync with focus session
- focus page only access

---

## Phase 5: Activity log

- render sessions in execution order
- filter by activity type
- day wise grouping
- readonly historical records

---

## Phase 6: Daily evaluation

- compute planned focus time
- compute completed focus time
- evaluate 60 percent completion rule
- store numeric daily results only

---

## Phase 7: Polish and closure

- ui cleanup
- edge case handling
- refresh recovery
- empty states
- error handling
- write readme
- add screenshots
- deploy to vercel or netlify

---

## Phase 8: React rewrite (post closure)

- rebuild same product using react
- use tailwind for styling
- preserve exact behavior
- improve component architecture

---

## Explicitly postponed

- coding metrics automation
- ide extensions
- browser extensions
- backend services
